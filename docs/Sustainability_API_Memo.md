# 持続可能性メモ（外部 API・コスト）

**目的**: [00_Flight_Academy_Strategy.md](00_Flight_Academy_Strategy.md) §3（持続可能性）に沿い、**ログイン後のみ**・**キャッシュ**・**従量見積り**の大方針を 1 ページに集約する。一次法令・公告の正本は当局サイトを正とする。

## 方針（2026-05-06）

| 領域 | 原則 |
|------|------|
| **認証境界** | 課金対象・高コストの外部呼び出しは**認証済み画面**に寄せ、匿名トラフィックは静的・ハブ中心に抑える。 |
| **キャッシュ** | 気象・NOTAM 系は既存の TTL / プロキシ設計に従い、**同一クエリのぶん回し**を避ける。詳細はコード（`api/`）と [02_System_Spec.md](02_System_Spec.md) の該当フロー。 |
| **キー管理** | API キーは **Vercel の環境変数** のみ。クライアントに出さない（[03_Development_Guide.md](03_Development_Guide.md)）。 |
| **観測** | 変更・障害は Sentry / Vercel / Supabase のログで追跡。商用条件の見直しは利用量の目視と [00](00_Flight_Academy_Strategy.md) §9 の判断軸に沿う。 |

**補足**: 本メモはドラフト。数値目標・契約条件の正本はサービス提供元のダッシュボードと Notion 等の運用を優先し、差分が出たらここを更新する。
