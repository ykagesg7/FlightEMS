## 🏗️ コンポーネント配置方針（ハイブリッド方式）

### 基本原則

- **共通コンポーネント**: `src/components`に配置（複数ページで使用されるコンポーネント）
- **ページ固有コンポーネント**: 各ページディレクトリの`components/`サブディレクトリに配置

### 配置判断基準

#### 共通コンポーネント（`src/components`に配置）

以下の条件を満たすコンポーネントは`src/components`に配置します：

1. **デザインシステムコンポーネント** (`ui/`)
   - Button, Card, Typography, Tabs など
   - プロジェクト全体で統一されたUI要素

2. **マーケティング共通コンポーネント** (`marketing/`)
   - RankBadge, MissionCard, ComingSoonBadge など
   - 複数のマーケティングページで使用

3. **MDX関連共通コンポーネント** (`mdx/`)
   - MDXLoader, MDXContent, CodeBlock など
   - 記事システム全体で使用

4. **共通ユーティリティコンポーネント** (`common/`)
   - ProgressRing など
   - 複数ページで使用される汎用コンポーネント

#### ページ固有コンポーネント（各ページディレクトリに配置）

以下の条件を満たすコンポーネントは各ページディレクトリに配置します：

1. **単一ページでのみ使用されるコンポーネント**
   - 例: `src/pages/articles/components/ArticleCard.tsx`
   - 例: `src/pages/gallery/components/PhotoGrid.tsx`

2. **ページの機能に密接に関連するコンポーネント**
   - 例: `src/pages/planning/components/flight/PlanningTab.tsx`
   - 例: `src/pages/test/components/QuizComponent.tsx`

3. **ページ固有のサブディレクトリ構造**
   - 例: `src/pages/articles/components/learning/` (ArticlesPageで使用)
   - 例: `src/pages/articles/components/seo/` (ArticleDetailPageで使用)

## 🔧 インポートパス規則

### 共通コンポーネントのインポート

共通コンポーネントは、ページの階層に応じて相対パスでインポートします：

```typescript
// src/pages/articles/ArticlesPage.tsx から
import { Button } from '../../components/ui/Button';
import { Card } from '../../components/ui/Card';
import RankBadge from '../../components/marketing/RankBadge';
import MDXLoader from '../../components/mdx/MDXLoader';
```

### ページ固有コンポーネントのインポート

ページ固有コンポーネントは、同じページディレクトリ内の相対パスでインポートします：

```typescript
// src/pages/articles/ArticlesPage.tsx から
import { ArticleDashboard } from './components/ArticleDashboard';
import { ArticleCard } from './components/ArticleCard';

// src/pages/articles/ArticleDetailPage.tsx から
import { CommentSection } from './components/CommentSection';
import { ReadingProgressBar } from './components/ReadingProgressBar';
```

### サブディレクトリ内のコンポーネントのインポート

```typescript
// src/pages/articles/ArticleDetailPage.tsx から
import { ArticleJsonLd } from './components/seo/ArticleJsonLd';
import { ArticleMetaTags } from './components/seo/ArticleMetaTags';

// src/pages/articles/components/ArticleDashboard.tsx から
import { AdaptiveLearningDashboard } from './learning/AdaptiveLearningDashboard';
```

## 📝 新規コンポーネント追加時のガイドライン

### 1. 配置場所の決定

新しいコンポーネントを作成する際は、以下の順序で判断します：

1. **複数のページで使用されるか？**
   - Yes → `src/components/` の適切なサブディレクトリに配置
   - No → 次のステップへ

2. **どのページで使用されるか？**
   - 単一ページ → そのページの`components/`ディレクトリに配置
   - 複数ページ → `src/components/` の適切なサブディレクトリに配置

3. **デザインシステムの一部か？**
   - Yes → `src/components/ui/` に配置
   - No → 次のステップへ

4. **マーケティング関連か？**
   - Yes → `src/components/marketing/` に配置
   - No → 次のステップへ

5. **MDX関連か？**
   - Yes → `src/components/mdx/` に配置
   - No → ページ固有コンポーネントとして配置

### 2. ファイル命名規則

- **コンポーネントファイル**: PascalCase（例: `ArticleCard.tsx`）
- **フックファイル**: camelCase with `use` prefix（例: `usePrevNext.ts`）
- **型定義ファイル**: camelCase（例: `types.ts`）
- **ユーティリティファイル**: camelCase（例: `articleIndex.ts`）

### 3. インポートパスの更新

新しいコンポーネントを追加したら、使用するすべてのファイルでインポートパスを更新します。

## ⚠️ 注意事項

### 共通コンポーネントの移動

共通コンポーネントをページ固有コンポーネントに移動する場合は、以下の点に注意してください：

1. **使用箇所の確認**: すべての使用箇所を確認し、インポートパスを更新
2. **型定義の確認**: 型定義が正しくインポートされているか確認
3. **テストの更新**: テストファイルのインポートパスも更新

### ページ固有コンポーネントの共通化

ページ固有コンポーネントを共通コンポーネントに移動する場合は、以下の点に注意してください：

1. **依存関係の確認**: ページ固有の依存関係がないか確認
2. **型定義の確認**: 型定義が適切にエクスポートされているか確認
3. **インポートパスの更新**: すべての使用箇所のインポートパスを更新
