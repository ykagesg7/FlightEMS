## 1. 北極星指標（NSM）：ALPM

### 1.1 名称と定義（合意ドラフト）

**ALPM** — *Active Learning Path Milestone*

- **意味**: ユーザーが **自分専用の学習ロードマップ**（パーソナライズされた学習経路）上で、**カレンダー週あたり 1 件以上**の **マイルストーン** を達成したことを数える指標。
- **マイルストーンの例**（実装時にデータモデルで列挙する）:
  - 指定された「苦手克服」単元の達成（診断・クイズ・読了の組み合わせはプロダクト定義）
  - ロードマップ上の **指定記事の読了**（`learning_contents` / 進捗イベントと対応させる）
- **狙い**: 潜在層には **自己効力感**（「自分でもできる」）、訓練生には **効率的な実力向上の実感**。中長期では **LTV（顧客生涯価値）** の最大化に寄与する行動を週次で促す。

### 1.2 計測契約（要決定）

実装・分析に入る前に **プロダクト・データで確定する項目**（現状は未確定のまま明記する）。

| 論点 | 要決定内容 |
|------|------------|
| **週の境界** | ユーザータイムゾーン（JST 固定か、アカウント設定か） |
| **単位** | 「週に ALPM を達成した **ユーザー数**」か「**達成イベント数**」か、または両方 |
| **同一週の複数達成** | 2 件目以降を NSM に含めるか、週1ユーザーあたりキャップするか |
| **デモ／内部アカウント** | GA4・DB 集計から除外するルール |
| **ロードマップ未設定ユーザー** | デフォルト経路（例: 気象・通信 PMF スライス）を ALPM の母集団に含めるか |

**正本の役割分担**: NSM の **定義・計測契約の文章**は **本書**。ダッシュボード用の **数式・イベント名・DB マッピング**は確定後 **[02_System_Spec.md](02_System_Spec.md)** に追記する（別タスク）。

## 3. 初期 PMF スライス：気象・通信

### 3.1 方針（CPL 主軸との関係）

- **リポジトリ全体のコンテンツ投資**は引き続き **CPL 記事を主軸**とし、[05_Content_Pipeline.md](05_Content_Pipeline.md) の Phase 2・マッピング精緻化を継続する。
- **プロダクト上の「最初の価値検証」**（PMF 確認、スモールスタート、チャーン抑制）として、**気象**と **通信** に **オンボーディング・ナビゲーション・診断コンテンツの既定**を寄せる。
  - **気象**: 既存の一般知識と結びつきやすく、**潜在層**の入口に向く。
  - **通信**: 「パイロットらしさ」を早い段階で体験でき、航法ほどの計算負荷が低く **離脱を抑えやすい**。

### 3.2 コンテンツ・ID

具体記事・単元の選定は **学習設計（ID）・執筆（しゃどーさん）** のアクションアイテム。確定後、本節または [05](05_Content_Pipeline.md) の該当週に **一覧表** を追記する。

---

## 4. AI の役割：コンシェルジュ（ナビゲーター）

- **採用する役割**: コンテンツの **大量自動生成の代替**ではなく、**一次情報（MDX 記事・運営者の本文）へユーザーを導くナビゲーター** に徹する。
- **根拠との整合**: 専門性と表現の公開粒度は **[00 §3.2](00_Flight_Academy_Strategy.md)**。AI は **根拠 URL・記事 ID** を踏まえた案内を優先し、ハルシネーションを前提にしない運用設計とする。

## 5. データ駆動：興味・学習ログ（適応学習の教師データ）

### 5.1 方針

- 初期段階から **興味関心** と **学習ログ**（記事進捗、クイズ、セッション等）を蓄積し、将来の **推奨付き選択モデル（アダプティブ・ラーニング）** への移行に備える。
- **既存基盤**（概念レベル）: `learning_progress`、`learning_sessions`、`user_learning_profiles`、`user_test_results`、`quiz_sessions` 等。詳細表は [docs/README.md](README.md) の DB 節。

### 5.2 実装の正本

- スキーマ追加・イベント命名・プライバシー境界の **仕様**は **[02_System_Spec.md](02_System_Spec.md)** および **[06_Long_Term_Execution.md](06_Long_Term_Execution.md)**（分析・GA4）に追記していく。
- 本書は **事業意図** の固定にとどめ、列定義のダブりを作らない。

## 6. 法務・信頼（CLO）：UX に載せる論点

**法的助言ではない**。利用規約・プライバシーポリシー案の **叩き台**は法務担当の別成果物とし、本リストは **プロダクト／コンテンツが触れるべき論点**のメモ。

| 論点 | UX への組み込みの例 |
|------|---------------------|
| **10 代の利用** | 保護者同意フロー、年齢に応じた表示制限の検討 |
| **実機訓練の非代替** | 診断・通信コンテンツの冒頭で **学科自学支援**であり実機・運航訓練を代替しない旨を明示 |
| **通信・電波・航空法** | **一般的な学科範囲の説明**に留める。免許・周波数・運用の具体的助言に読まれない表現、**電波法・航空法上の助言と誤解されない**工夫 |
| **AI 出力** | ナビゲーターは **記事根拠**を示す。誤情報を防ぐための固定免責（画面テンプレートは別タスク） |

詳細チェックは **[00 §3.3–3.4](00_Flight_Academy_Strategy.md)** と併覧。
